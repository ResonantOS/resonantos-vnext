# Legacy Documentation

This directory contains historical documentation preserved for contextual reference. Documents here are not actively maintained and may describe states of the codebase that no longer exist.

## Contents

- `reports/` — historical reports (QA, security, red-team, browser integration).
- `planning/` — historical planning, task specifications, and migration plans.
- `qa/` — historical QA and testing reports.
- `experiments/` — experimental and speculative documentation.

## Policy

- No document in legacy/ should be deleted without explicit archival justification.
- Legacy documents are preserved for archaeological value, not as current-state truth.
- If information from a legacy document is needed in active docs, extract it, verify it against the current codebase, and cite the source.
- Do not update legacy documents to match current state — they capture a moment in time.

# Legacy QA Reports

This directory contains historical QA (Quality Assurance) reports preserved for contextual reference. These documents capture testing and validation activity at specific points in time and may describe resolved issues or outdated test scenarios.

## Why These Are Preserved

- **Testing evidence** — record of QA activity performed.
- **Issue archaeology** — understanding when and how specific problems were identified.
- **Platform validation history** — cross-platform testing documentation.

## Current Contents

- `QA-BROWSER-FINAL-REPORT.md` — final browser QA report.
- `QA-BROWSER-REPORT.md` — browser QA report.
- `QA-BUILD-REPORT.md` — build system QA report.
- `QA-PLATFORM-REPORT.md` — platform compatibility QA report.
- `QA-RESONATOR-FINAL-REPORT.md` — Resonator QA report.
- `QA-SECURITY-REPORT.md` — security QA report.

## Usage

- Consult these when investigating historical QA findings.
- Do not treat these as current testing evidence — re-run validation commands for current-state verification.
- If findings here describe unresolved issues, verify they still apply against the current codebase.
